## Environment variables

Copy `.env.example` to `.env` (never commit secrets):

```bash
cp .env.example .env
```

| Variable | Format | Purpose |
|----------|--------|---------|
| `VITE_SITE_URL` | `https://www.sansolution.tech` | Canonical site URL |
| `VITE_CONTACT_ENDPOINT` | `/api/contact` (default) | Contact form POST destination |
| `VITE_CONTACT_EMAIL` | `kontak@sansolution.tech` | Display / mailto email |
| `VITE_WHATSAPP_NUMBER` | Digits only, e.g. `628568862327` | WhatsApp Business number |
| `VITE_GA_MEASUREMENT_ID` | `G-XXXXXXXXXX` | Optional GA4 ID |
| `RESEND_API_KEY` | Resend secret key | **Server-only** — sends form emails |
| `CONTACT_TO_EMAIL` | `kontak@sansolution.tech` | Inbox for form submissions |
| `RESEND_FROM_EMAIL` | `SAN Solution <kontak@sansolution.tech>` | Verified Resend sender |

### Contact form setup (Resend)

1. Create an account at [resend.com](https://resend.com) and verify the `sansolution.tech` domain.
2. Create an API key and set `RESEND_API_KEY` in Netlify (Site settings → Environment variables) and in local `.env`.
3. Set `CONTACT_TO_EMAIL=kontak@sansolution.tech` and a verified `RESEND_FROM_EMAIL`.
4. Production uses Netlify Function at `/api/contact` → `/.netlify/functions/contact`.
5. Local `npm run dev` serves the same path via Vite middleware.
6. Success UI appears **only** after Resend accepts the message.
7. Spam protection: honeypot + minimum form-fill time (client and server).

Never expose `RESEND_API_KEY` with a `VITE_` prefix.

## SEO configuration

- **Canonical domain:** `https://www.sansolution.tech`
- Apex `https://sansolution.tech` should permanently redirect to www (configure at DNS/hosting).
- Indonesian is the default locale (`/`); English lives under `/en/…`.
- Hreflang is set per page via `SEOHead`.
- Sitemap is generated at build time and referenced in `robots.txt`.
- Structured data: Organization on homepage; FAQPage where FAQ is visible; BreadcrumbList on deeper pages.

See `docs/seo-rendering.md` for SPA prerender recommendations.

---

## Domain configuration

Recommended hosting redirects:

```text
https://sansolution.tech/*  →  301/308  →  https://www.sansolution.tech/*
```

SPA fallback: all unknown paths should serve `index.html` with correct HTTP status for 404 handled client-side (or configure edge 404 if using prerender later).

Example Netlify `_redirects` is included in `public/_redirects`.

## Deployment

1. Set environment variables in the host.
2. Run `npm run build`.
3. Deploy the `dist/` folder.
4. Verify:
   - Apex → www redirect
   - `/robots.txt` and `/sitemap.xml`
   - Contact form with real endpoint
   - `/en/` homepage and hreflang tags

## Contribution notes

- Preserve factual portfolio, timeline, and client context — do not invent testimonials, metrics, or partnerships.
- Prefer business-clear copy over engineering jargon in visitor-facing UI.
- Keep contact success paths honest (no simulated submissions).
- Indonesian content is the default; keep EN translations in sync when editing copy.
